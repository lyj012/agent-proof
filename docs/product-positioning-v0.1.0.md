# AgentProof v0.1.0 产品定位文档

## 1. 项目定位

AgentProof 是一个在 Windows 本地运行的 AI 辅助开发交付报告生成器。

它面向使用 Codex、Claude Code 等 AI 编程工具完成开发任务的个人开发者，帮助他们把 Git 变更、AI 开发记录、构建测试结果、风险说明和人工审核信息整理成一份客户或项目负责人能够阅读的 Markdown 交付报告。

AgentProof 解决的核心问题不是“如何使用 AI 写代码”，而是“使用 AI 完成开发后，如何清楚地证明本次交付包含什么、验证过什么、还存在哪些风险”。

## 2. 核心用户

v0.1.0 只服务一种核心用户：

使用 AI 编程工具完成项目开发，并需要向客户或项目负责人提交开发成果的个人开发者。

典型用户包括：

- 自由职业开发者；
- 外包开发者；
- 独立开发者；
- 小团队中的实际开发人员。

v0.1.0 不优先服务：

- 大型企业审计部门；
- 多人研发管理团队；
- 非技术客户直接操作；
- SaaS 平台管理员。

## 3. 使用场景

开发者完成一个开发任务后，交付证据通常分散在多个位置：

- Git commit；
- Git diff；
- 修改文件列表；
- Codex 或 Claude Code 开发记录；
- 构建命令结果；
- 测试结果；
- 风险说明；
- 人工检查结论。

这些信息如果手工整理，容易遗漏测试、风险、验收步骤和人工确认项，也难以让客户快速理解本次交付是否完整。

AgentProof 的 v0.1.0 目标，是把这些分散信息整理成结构固定、客户可读、便于人工审核的 Markdown 报告。

## 4. 产品价值

### 4.1 对开发者

- 降低整理交付说明的时间成本；
- 避免遗漏测试结果、风险说明和验收步骤；
- 形成标准化、可复用的交付流程；
- 为自由职业、外包和远程项目提供更专业的交付材料；
- 将 AI 辅助开发过程转化为可交付、可复盘的项目证据。

### 4.2 对客户或项目负责人

- 快速理解本次任务完成了什么；
- 查看本次涉及的 commit、修改文件和 diff 摘要；
- 了解构建与测试状态；
- 看到风险、遗留问题和人工审核项；
- 按照明确步骤完成验收。

## 5. 产品边界

AgentProof 是：

- 交付证据整理工具；
- 交付报告生成工具；
- 人工审核辅助工具；
- AI 工程化交付流程的落地验证项目。

AgentProof 不是：

- 代码质量认证机构；
- 自动审计系统；
- 自动验收系统；
- AI 代码正确性证明系统；
- 项目管理平台；
- AI 使用量统计工具。

AgentProof 生成报告，不代表代码一定正确，也不代表需求一定完成。最终结论仍需要开发者或项目负责人进行人工审核。

## 6. 与 ai-usage-tool 的区别

ai-usage-tool 面向开发者本人，主要用于查看 AI 工具使用记录、统计使用量和进行个人开发复盘。

AgentProof 面向客户或项目负责人，主要用于说明完成了什么、展示代码变更证据、展示构建和测试结果、说明风险和遗留问题，并提供验收步骤。

两个项目独立，不共享定位，也不合并代码。

## 7. v0.1.0 输入范围

### 7.1 Git 仓库

用户明确指定一个本地 Git 仓库路径。程序读取：

- 仓库有效性；
- 最近一次 commit；
- commit message；
- 修改文件列表；
- diff 摘要或 diff 内容；
- 新增和删除行统计。

### 7.2 任务信息

用户通过 `task.json` 提供任务信息，包括：

- 任务名称；
- 原始需求；
- 构建结果；
- 测试结果；
- 已知风险；
- 遗留问题。

### 7.3 AI 开发记录

用户明确指定一个 Codex 或 Claude Code 开发记录文件。

v0.1.0 不自动扫描整台电脑，不读取全部历史记录，也不尝试自动识别所有 AI 工具的数据格式。第一阶段只要求支持一种经过真实验证的记录格式。

### 7.4 输出路径

用户指定 Markdown 报告输出位置。默认输出文件名为：

```text
delivery-report.md
```

## 8. v0.1.0 输出范围

AgentProof v0.1.0 生成一份 Markdown 交付报告，至少包含：

1. 交付概览；
2. 原始需求；
3. Git commit；
4. 修改文件；
5. diff 摘要；
6. AI 辅助开发记录摘要；
7. 构建结果；
8. 测试结果；
9. 已知风险；
10. 遗留问题；
11. 人工审核清单；
12. 客户验收说明；
13. 证据限制说明；
14. 报告生成时间和 AgentProof 版本。

报告必须让读者能够区分“用户声明的信息”和“AgentProof 实际读取到的证据”。

## 9. P0 功能范围

v0.1.0 的 P0 功能包括：

- 本地 CLI：支持 `agentproof --help`、`agentproof --version`、`agentproof generate`；
- Git 信息读取：验证 Git 仓库，读取最近一次 commit、message、修改文件和 diff 摘要；
- 任务文件读取：读取并校验 `task.json`；
- 开发记录读取：读取用户指定的 Codex 或 Claude Code 记录文件；
- Markdown 报告生成：输出结构固定、客户可读的交付报告；
- 人工审核清单：提醒开发者确认需求一致性、修改范围、敏感信息、构建测试、风险和验收步骤；
- 敏感信息保护：默认不直接输出完整提示词、完整 AI 回复、API key、token、密码、cookie、Authorization、环境变量值和不必要的本机绝对路径。

## 10. 明确不做

AgentProof v0.1.0 不做：

- 登录注册；
- 数据库；
- Web 页面；
- 前后端系统；
- 云端服务；
- SaaS；
- 多租户；
- 团队权限；
- 支付；
- GitHub App；
- GitHub OAuth；
- MCP Server；
- AI 模型调用；
- 自动生成代码；
- 自动修改代码；
- 自动判断代码是否正确；
- 自动判断需求是否完成；
- 自动执行构建命令；
- 自动执行测试命令；
- AI token 和费用统计；
- PDF 或 Word 导出；
- 多套报告模板；
- 数据可视化。

这些非目标用于保护 v0.1.0 的最小闭环，避免项目过早扩展成平台型产品。

## 11. 证据可信度规则

AgentProof 报告必须区分以下证据状态：

### 11.1 开发者声明

由用户在 `task.json` 中填写的信息，例如测试结果为 `passed`。

AgentProof 可以展示该信息，但不得把它描述为 AgentProof 已验证通过。

### 11.2 证据文件存在

AgentProof 找到了用户指定的测试、构建或开发记录文件。

这只能说明文件存在并被读取，不等于文件内容真实、完整或可代表最终状态。

### 11.3 AgentProof 实际执行

v0.1.0 不执行构建和测试，因此报告中不得声称 AgentProof 已执行构建或测试。

### 11.4 人工审核确认

最终交付结论需要开发者或项目负责人完成检查。AgentProof 的职责是提供清晰的审核材料和检查清单，而不是替代人工判断。

## 12. 人工审核清单

报告应包含以下人工审核项：

- 原始需求与交付内容是否一致；
- 修改文件是否属于本次任务范围；
- 是否存在无关代码修改；
- 是否包含密钥、token、密码、cookie 或其他敏感信息；
- 构建结果是否由开发者确认；
- 测试结果是否由开发者确认；
- 风险和遗留问题是否已经说明；
- 客户验收步骤是否明确、可执行；
- 报告中的证据限制是否已经告知客户或项目负责人。

## 13. 成功标准

AgentProof v0.1.0 满足以下条件即可发布：

- 可以在 Windows 本地安装；
- CLI 可以正常运行；
- 可以读取一个真实 Git 仓库；
- 可以读取最近一次 commit 和 diff 摘要；
- 可以读取一个真实 AI 开发记录；
- 可以读取任务 JSON；
- 可以生成完整 Markdown 报告；
- 报告包含风险和人工审核清单；
- 敏感信息不会直接进入报告；
- 自动化测试真实通过；
- 使用 AgentProof 项目自身完成一次真实报告生成；
- 发布 GitHub Release v0.1.0。

## 14. 与 ai-engineering-delivery 的关系

AgentProof 是用于验证 `ai-engineering-delivery` 的真实案例项目。

完整流程是：

```text
使用 ai-engineering-delivery 分析 AgentProof
-> 明确需求和范围
-> 开发 AgentProof
-> 进行代码审查和测试
-> 生成 AgentProof 自身的交付报告
-> 记录 Skill 的有效点和缺陷
-> 把完整案例写入 ai-engineering-delivery
```

AgentProof 的首要价值，是证明这套 AI 工程化交付方法能够真正指导一个项目从需求走到发布。

商业化不是 v0.1.0 的目标。
